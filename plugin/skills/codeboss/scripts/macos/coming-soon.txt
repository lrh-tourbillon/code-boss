macOS support is planned but not yet implemented.

The macOS version will need equivalent scripts for:
- dispatch (bash/zsh equivalent of dispatch.ps1)
- run-phase (bash/zsh equivalent of run-phase.ps1)
- send-claude-message (macOS accessibility/AppleScript equivalent of Send-ClaudeMessage.ps1)

Key differences from Windows:
- Shell scripts (.sh) instead of PowerShell (.ps1)
- macOS Accessibility API or AppleScript instead of Windows UI Automation
- ~/Library/Application Support/ instead of %APPDATA%
- Different process management (launchd vs Start-Process -WindowStyle Hidden)

Contributions welcome.
